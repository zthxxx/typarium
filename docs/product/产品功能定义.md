# 产品功能定义

本文面向功能验收：每个功能给出行为定义与验收标准。需求背景见[原始诉求](../requirements/原始诉求.md)，实现方案见[功能技术设计](../design/功能技术设计.md)

## 页面结构

```
┌─────────────────────────────────────────────────┐
│ Header: logo | 语言选择器(TS) | i18n 切换 | 分享  │
├──────────────────────────┬──────────────────────┤
│ 预设类型按钮条            │                      │
│┌────────────────────────┐│   代码编辑器 (monaco) │
││                        ││                      │
││   Euler 可视化画布      ││                      │
││                    [any badge 默认悬浮于边界]   │
│└────────────────────────┘│                      │
├──────────────────────────┴──────────────────────┤
│ Footer: 项目信息 | TS 版本号 | GitHub 链接        │
└─────────────────────────────────────────────────┘
```

- 桌面端左右分栏（画布 : 编辑器 ≈ 6 : 4，可拖拽调整）；窄屏（< 768px）上下堆叠，画布在上
- 画布不可缩放、不可平移，保持最小尺寸（宽 ≥ 360px），视口不足时页面滚动
- Footer 明示引擎所用 TypeScript 版本号（原因见 [ADR-0002](../adr/0002-技术栈-tanstack-start-与-typescript-单版本-pin.md)）

## 编辑器管线

行为定义：

1. 用户在 monaco 编辑器输入 TypeScript 代码，strict 常开
2. 编辑器实时给出语法与类型诊断，错误位置波浪线高亮 + hover 显示错误信息，形式等同 VS Code
3. 存在任何 error 级诊断时，可视化保持上一次成功的结果，画布角落显示「等待代码修复」的轻提示
4. 诊断全部通过后，1.2s 内无新输入则自动提交可视化；期间有新输入则重新计时
5. 提交进行中再次触发提交，旧任务作废（以最新代码为准）

验收标准：

- 输入含 `unknwon` 拼写错误的代码 → 编辑器在该标识符处报 `Cannot find name` 且画布不更新
- 修复错误后停止输入，1.2s 后画布自动更新
- 连续输入期间画布不闪烁、不中途更新

## 可视化元素的确切表现

- **`unknown`**：铺满画布的最外层区域，始终存在（即使代码为空）
- **`never`**：`∅` 字形点阵，均匀铺在所有集合区域内（含空画布），类比透明背景棋盘格；hover 点阵显示「never 是空集，被所有集合包含」的说明
- **`any`**：不进入 Euler 图。画布与编辑器边界处悬浮一个斜置约 15° 的橙红圆角 badge，带投影阴影（图内集合无投影，以此区分层级）；用户可拖拽到屏幕任意位置，位置持久化；hover 显示「any 不属于集合范畴」说明与当前解析为 any 的导出清单；代码里没有 any 时 badge 不出现
- **字面量类型**：所属基底域内的小环 + 标签（如 `"foo"` 在 `string` 域内）
- **union**：包住全部成员区域、中间收腰的连续轮廓
- **包含**：嵌套轮廓，层间留 padding，视觉上一眼读出「谁在谁里面」
- **等价类**：相互可赋值的导出合并为一个区域，标签并列展示
- **重叠未知**：引擎无法判定相交性的类型对，重叠部分用虚线边界呈现，hover 说明「TypeScript 无法归约这对类型的交集，工具不猜测」
- 每个集合分配独立的边框色 + 淡色半透明花纹背景（色板见 [ADR-0009](../adr/0009-色彩系统-semi-color-tag.md)）；边框有呼吸式波动动效，`prefers-reduced-motion` 时静止

## 预设类型按钮

- 画布顶部固定按钮：`any` `unknown` `object` `void` `undefined` `null` `never` `{}`
- 点击 = 向编辑器末尾插入一行真实代码（如 `export type _object = object`），再次点击删除该行；按钮选中态与代码内容双向同步
- 代码是唯一事实源：用户手动删除该行，按钮回到未选中态

验收标准：点击 `object` 后画布出现 object 域轮廓且编辑器出现对应代码行；手动删除该行后按钮取消选中

## hover 类型推断

- 画布上 hover 集合标签：展开该类型的推断结果（展开后的规范类型文本），样式仿 VS Code hover 卡片
- 编辑器内 hover：monaco 原生 quickInfo
- 两处文案同源（同一分析引擎），不允许出现两边推断结果不一致

## 持久化与分享

- 所有用户输入自动保存到 IndexedDB（编辑停顿 500ms 后写入），刷新 / 重新进入页面完整恢复：代码、语言偏好、any badge 位置
- 首次访问（无本地存档、无分享参数）加载「协变示例」预设代码
- 分享链接：Header 分享按钮提供「带当前内容 / 仅站点链接」两个选项；带内容时把代码编码进 URL hash（URL-safe Base64 变体，带 schema 版本号）
- `Cmd / Ctrl + S`：等价于「带内容分享」，链接写入剪贴板 + toast 提示
- 打开带内容的分享链接：内容载入编辑器并立即可视化，同时 toast 提示「已加载分享内容」；本地已有存档时分享内容优先，原存档在用户下一次编辑前不覆盖

优先级规则：URL hash 内容 > IndexedDB 存档 > 默认示例

## i18n

- 中文 / English 两种界面语言，Header 切换
- 默认读 `navigator.language`（zh 前缀 → 中文，否则 English）
- 切换后写入 IndexedDB 持久化；语言偏好不进分享链接（个人偏好不应强加给接收者）

## 期 1 验收清单

[原始诉求](../requirements/原始诉求.md#教学演示用例)中的教学演示用例逐条输入，图形正确呈现：

- [ ] 协变：`CoNarrow ⊂ CoWide` 嵌套呈现
- [ ] 逆变：`ContraFromWide ⊂ ContraFromNarrow` 方向翻转呈现
- [ ] 函数参数逆变：`FnWideParam ⊂ FnNarrowParam`
- [ ] 方法双向协变：`Kennel` 与 `DogKennel` 合并为等价区域 + 偏差标注；`StrictKennel ⊂ StrictDogKennel`
- [ ] UnionToIntersection：交集嵌于两成员重叠处
- [ ] Tagged Union：三 variant 两两不相交、`RowData ⊂ LooseRowData` 真包含
- [ ] `{}`：覆盖除 `null` / `undefined` 外的一切
- [ ] 预设按钮 8 个全部可用
- [ ] any badge：出现、拖拽、hover 说明、位置持久化
- [ ] never 点阵铺满各集合区域
- [ ] 持久化：刷新恢复；分享：带内容链接在无痕窗口打开可完整还原
- [ ] i18n 中英切换生效且持久化
- [ ] 空代码时画布仍呈现 unknown 全集 + never 点阵

## 非目标（本期不做）

- 语言切换的实际生效：Header 选择器只有 TypeScript 一项，多语言只保留架构扩展点
- 任意复杂对象格局的精确判空：超出可判定核心的类型对进「重叠未知」态，不猜测
- 画布缩放 / 平移
- enum 名义语义、class 声明的精确处理（按对象类型近似）
- 移动端的编辑体验优化（保证最小尺寸可读即可）
- 账号体系、云端存储：分享即 URL，存储即本地
