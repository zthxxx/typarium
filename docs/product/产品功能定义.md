# 产品功能定义

本文面向功能验收：每个功能给出行为定义与验收标准。需求背景见[原始诉求](../requirements/原始诉求.md)（含第二轮修订），实现方案见[功能技术设计](../design/功能技术设计.md)

## 页面结构

```
┌─────────────────────────────────────────────────┐
│ Header: logo | 语言选择器(TS) | i18n 切换 | 分享  │
├─────────────────────────────┬───────────────────┤
│ 预设类型 chip 条（可多行）    │ ← 拖拽手柄        │
│┌───────────────────────────┐│                   │
││                           ││  编辑器抽屉        │
││   矩形包含画布（流式充满）  ││  (monaco)         │
││          [any badge 默认悬浮于边界]             │
│└───────────────────────────┘│                   │
├─────────────────────────────┴───────────────────┤
│ Footer: 类型分析引擎版本                          │
└─────────────────────────────────────────────────┘
```

- 页面宽度自适应全屏：画布流式充满可用空间，无固定宽高比、无横向滚动
- 桌面（≥ 1024px）：编辑器为内联右栏，左缘拖拽调宽（320px 至视口 70%），标题栏按钮可完全收起（收起后右缘留一条细轨用于恢复）
- 窄屏（< 1024px）：编辑器为右侧浮出抽屉，盖在画布上；收起时整体移出屏幕，右缘保留把手按钮 —— 不采用上下堆叠
- 画布不可缩放不可平移，最小逻辑尺寸 320×240
- Footer 只保留引擎版本信息，无 GitHub 链接

验收标准：拖动浏览器窗口宽度，画布与编辑器实时回流无横向滚动条；1024px 以下编辑器变浮出抽屉，点收起立即回缩、画布无需滚动即可完整可见

## 可视化：矩形包含布局

行为定义（完整规则见 [ADR-0012](../adr/0012-可视化范式-矩形包含布局.md)）：

- 每个展示实体是一个圆角矩形：粗描边 + 淡色半透明填充，顶部一行标签
- 只表达包含或不包含：包含 = 矩形嵌套（层间有 padding），无包含关系 = 并列矩形；不存在部分重叠的图形形态
- 兄弟实体在容器内平分空间：槽位奇数（> 1）向上取偶再做 cols ≥ rows 的均衡网格，多余格子留白
- 实体容器内恒有一个 other 留白槽，表达「除子集外还有别的值」；隐式画布根没有 other 槽
- 等价实体（互为子集）合并为同一矩形，每多一个成员加一圈内缩边框，标签以 `≡` 并列
- **空画布**：代码无导出且无激活预设时，画布完全为空（只有一句引导文案），没有任何默认图形

具体布局验收用例：

| 输入                          | 预期                                   |
| ----------------------------- | -------------------------------------- |
| 单个 `export type A = string` | A 占满整个画布区域                     |
| 两个互不包含                  | 左右平分（2×1）                        |
| 三个互不包含                  | 2×2 网格占三格，余一格留白             |
| `A ⊃ B`                       | B 占 A 内容区一半，另一半留白（other） |
| `A ⊃ {B, C}`，B C 互不包含    | A 占满画布，B、C 与 other 走 2×2 网格  |
| `A ≡ B`                       | 单矩形双边框环，标签 `A ≡ B`           |
| `A ⊃ B ⊃ C`                   | 三层嵌套，逐层 padding                 |

## 特殊类型

- **`unknown`**：被展示时画布本身就是它 —— 画布边框转品牌蓝强调，左上角 `unknown ≡ ...` 标签；不单独占一个矩形
- **`never`**：被展示时整个画布背景铺 `∅` 点阵（类比透明背景棋盘格）+ 左下图例；hover 背景空白区域（含矩形间隙、圆角边缘）时 tooltip 列表末尾出现 `∅ never` 行
- **`any`**：不进入图内。橙红斜置、带投影的可拖拽悬浮 badge，默认横跨画布与编辑器边界；hover 显示说明与当前解析为 any 的清单。预设区的 any chip 无论是否激活都保持暖红警示样式
- **tooltip 堆叠列表**：hover 画布任意位置，浮出包含该点的实体列表（外层在前，每行「色块 + 名字 + 类型文本」）；多个类型叠放时这是唯一的逐层读法

验收标准：点 never 预设 → 背景点阵 + 图例出现；hover 两层嵌套矩形的内层 → tooltip 自外向内列出两个实体各一行

## 预设类型区

- 目录 26 项按分类排序展示，不渲染分组样式：8 基本（string/number/boolean/bigint/symbol/null/undefined/object）→ 4 内置（unknown/never/void/any）→ 3 常见（`Array<T>`/`Object`/`Function`）→ 11 代码模板
- 前 15 项（基本 + 内置 + 常见）为 **virtual 预设**：必须全部直接可见（chip 条可多行换行，禁止收进浮层）；点击 = 画布展示/取消该类型，编辑器代码零变化
- 11 个代码模板为 **snippet 预设**：收进「代码模板」浮层；点击 = 编辑器追加 `export type CN = <模板>`，N 自动递增，与上文之间保证一行独立空行

验收标准：

- 点 `string` chip → 画布出现 string 矩形、编辑器内容不变、chip 转激活态；再点取消
- 连点两个模板 → 编辑器末尾出现 `export type C1 = ...` 空行 `export type C2 = ...`
- 窗口缩窄时 virtual chip 换行仍全部可见

## 编辑器管线

1. monaco 输入，strict 常开，语法与类型错误在编辑器内波浪线高亮（monaco 内置 TS worker）
2. 语义分析引擎为 TypeScript 6.0.3，Footer 明示；编辑器诊断与引擎判定出现版本差时以引擎为准
3. 停止输入 1.2s 后自动提交可视化；存在 error 级诊断时画布保持上一次成功结果
4. virtual 预设切换立即触发分析，不等防抖
5. 编辑器光标落在某导出声明内时，画布对应矩形高亮（双向联动）

验收标准：输入 `export type X = unknwon` → 编辑器报错、画布不变；改回正确拼写停 1.2s → 画布更新

## 持久化与分享

- 代码与激活的 virtual 预设都写 IndexedDB（键入 300ms 防抖），刷新完整恢复
- 分享链接可选携带内容：envelope 含 `{ code, presets }`；`Cmd / Ctrl + S` 复制带内容链接 + toast
- 恢复优先级：URL hash > IndexedDB > 默认示例；老版本链接（无 presets 字段）照常解码
- locale 存 localStorage，不进分享链接

## 图类型选择器（revision 3）

Presets 下方、画布上方一行：「图类型」标题 + 淡灰 info icon + 「Euler」「Hasse」radio

- 状态两层：用户选择与生效选择。从未选择过默认 Euler；Euler 无法如实绘制当前包含结构时自动禁用该选项、显示并生效 Hasse；代码变化使 Euler 恢复可画且用户未手动选过 Hasse 时，选项与展示自动回到 Euler；只有手动选择 Hasse 后才固定保持、不再自动恢复。用户选择不跨会话持久化
- 标题 + info icon 整体为 hot area，hover 出大 Popover：分别解释两种图型，末尾两张并排示例图用真实布局引擎渲染固定输入（C1 包含 C2 与 C3，C2 与 C3 无交集），保证示例与实际渲染永远一致

## 双向高亮（revision 3）

- 编辑器 → 画布（已有）：光标落在导出声明内，对应图形高亮、其余半透明；该导出被删除后高亮立即消失（派生自光标位置与最新分析结果）
- 画布 → 编辑器（新增）：hover 图形时，其等价类中每个来自代码的导出对应的声明行整行黄色背景高亮；一个图形可能对应多个相等导出，全部行同时高亮；离开画布即清除

## 启动进度（revision 3）

- 画布区域显示真实进度条与当前环节小字（下载类型引擎 / 启动引擎 / 恢复内容 / 计算首图），进度来自真实信号（下载字节与阶段完成），不做时间动画
- 加载完成即见内容：首次访问显示示例代码的预计算图形；有本地缓存或 URL hash 时恢复对应内容与图形（缓存按引擎版本 + 代码 + 预设精确匹配，引擎就绪后重验证）；引擎未就绪期间画布已可视，编辑器（monaco）在首次分析落地后再加载
