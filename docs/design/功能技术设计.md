本文是实现层的技术方案，覆盖语义引擎、布局引擎、渲染、预设、编辑器管线、持久化与分享。架构分层与 IR 契约见[工程架构设计](../architecture/工程架构设计.md)，行为验收见[产品功能定义](../product/产品功能定义.md)

## 沿革

当前方案是第二轮需求（2026-07-16，见[原始诉求](../requirements/原始诉求.md#第二轮需求2026-07-16)）后的 v2 形态，语义引擎随后按 [ADR-0015](../adr/0015-统一单一-typescript-资源-6.0.3.md) 统一为 typescript@6.0.3 单一资源（tsgo-wasm 探针架构使用一轮后退役，工程经验存 [ADR-0013](../adr/0013-引擎切换-tsgo-wasm.md)）。可视化范式反转记录见 [ADR-0012](../adr/0012-可视化范式-矩形包含布局.md)；v1 期间沉淀的 checker 实证结论（lazy 归约、双向协变塌缩等）依然有效并继续被引擎使用

## 语义引擎（adapters/typescript，运行于 worker）

### 架构：单一 6.0.3 worker 供给画布与编辑器

typescript@6.0.3（exact pin，JS 实现最后过渡线）+ @typescript/vfs 起 LanguageService，一个 worker 五个方法（comlink expose）：

- `analyze(source, virtualTypes)` —— 画布语义：export 扫描 → checker.getTypeAtLocation → special 分类（any 用 TypeFlags、never 用 `isTypeAssignableTo(T, getNeverType())` 避 lazy 归约）→ 非 special 两两 `isTypeAssignableTo` 双向查询出 equivalent / subset / superset / unrelated；虚拟预设合成进 `/virtual.ts`（`export type __VN = <typeText>`），实体 id `preset:<label>`
- `check(source)` —— 350ms 防抖的快速诊断（syntactic + semantic），驱动编辑器波浪线
- `quickInfo(source, offset)` —— LSP 风格 hover 文本，编辑器 hover 与画布 tooltip 同源
- `completions(source, offset)` —— 补全条目（上限 60）
- `dispose()`

`TypeEntity.expandedText` 由 `typeToString(type, undefined, InTypeAlias | NoTruncation)` 生成（alias 一层展开），tooltip 优先展示它 —— 这是 hover 类型展开的教学载荷。用户代码存在 error 诊断时 analyze 返回空实体 + 诊断，画布保持上一次成功结果

**monaco 侧不加载任何 TS worker**（ADR-0015）：编辑器只装 monarch 语法高亮，诊断 markers、hover provider、completion provider 全部接到上述 worker —— 运行时只有一份 TypeScript 实现

### export 扫描

同包 parser（`createSourceFile` 语法层）：

- 纳入顶层带 `export` 修饰符的 type alias / interface / enum；带类型参数的仅当全部参数有默认值时纳入（裸引用即默认实例化）
- `typeText` 取源码切片（alias 取 RHS、interface/enum 取整个声明），压缩空白单行化，过长截断加 `…`
- 实体上限 24，超限截断并附 warning 诊断

## 布局引擎（core/layout）

纯函数 `computeRectLayout({ entities, relations, viewport }) → RectLayoutResult`，规则见 [ADR-0012](../adr/0012-可视化范式-矩形包含布局.md)：

1. 分流：`universe` 实体进 `universeIds`（画布本身）、`empty` 进 `emptyIds`（背景点阵）、`outside-set-theory` 排除（badge）
2. 等价类 union-find 合并（按声明序定根，对 relations 顺序不敏感）；一个类一个 `EntityRect`，`ringCount` = 类大小
3. 包含森林：parent = 最小真超集，多个最小取声明序最早；检测到恶性环时提升最早成员到顶层并写 warning
4. 递归网格平分：隐式画布根槽位 = 顶层类数；实体容器槽位 = 子类数 + 1（other 槽）；槽位 1 占满、奇数（> 1）向上取偶、双因子分解 cols ≥ rows 且差最小；子项声明序 row-major 填格，空格留白
5. 容器过小放不下子级时跳过子级并写 warning（深嵌套退化保护）

### 几何常量口径

渲染层与布局共用同一份常量（`core/layout/constants.ts`）：

| 常量           | 值      | 含义                             |
| -------------- | ------- | -------------------------------- |
| `CANVAS_PAD`   | 14      | 画布内边距                       |
| `CELL_GAP`     | 12      | 网格单元间隙（背景从这里透出）   |
| `RING_INSET`   | 5       | 等价类每一圈边框的内缩量         |
| `LABEL_STRIP`  | 26      | 矩形顶部标签行高                 |
| `CONTENT_PAD`  | 10      | contentBox 相对 outer 的额外内缩 |
| `RECT_RADIUS`  | 14      | 圆角半径                         |
| `MIN_VIEWPORT` | 320×240 | 画布最小逻辑尺寸                 |

`contentBox = outer 内缩 (ringCount × RING_INSET + CONTENT_PAD)，顶部额外 + LABEL_STRIP`；viewport 由渲染层 ResizeObserver 实测传入，布局全程确定性、零随机

## 渲染（views/RectCanvas）

div 绝对定位渲染（放弃 SVG：矩形 + 文本标签用 CSS 表达力更好），画布容器流式充满可用空间：

- **矩形**：圆角 + 3px 实色描边（semi-color-tag 色板按声明序取色相，[ADR-0009](./../adr/0009-色彩系统-semi-color-tag.md)）+ 淡色半透明填充；等价类的额外成员画内缩边框环；hover / 编辑器光标命中时该矩形高亮、其余降透明度
- **never 点阵**：`canvas-dots` CSS 工具类（`radial-gradient` 平铺），never 被展示时铺满画布背景 + 左下图例 chip；未展示时画布是素色画板
- **unknown**：被展示时画布边框转品牌蓝强调 + 左上 `unknown ≡ ...` 标签
- **空画布**：无任何可绘实体时只显示一句引导文案（点预设或在编辑器 export）
- **堆叠 tooltip**：hover 画布任意点，列出包含该点的实体清单（外层在前，每行色块 + 名字 + typeText），末尾附 `∅ never` 行；跟随光标、近右缘翻转方向
- **any badge**：沿用 v1 —— 橙红斜置、投影、可拖拽，默认横跨画布与编辑器边界

## 预设（PresetService + PresetsBar）

目录 26 项，顺序即展示顺序，不渲染分组样式（[ADR-0014](../adr/0014-预设双轨与编辑器抽屉.md)）：

- **基本（virtual）**：`string` `number` `boolean` `bigint` `symbol` `null` `undefined` `object`
- **内置（virtual）**：`unknown` `never` `void` `any` —— any 的 chip 常驻暖红警示色
- **常见（virtual）**：`Array<T>`（分析为 `Array<unknown>`）`Object` `Function`
- **模板（snippet）**：`string & number`、`string | number`、`string | number | boolean`、`() => string`、`() => number`、`() => string | number`、`(_: string) => void`、`(_: number) => void`、`(_: string | number) => void`、`<T extends string>(_: T) => void`、`<T extends string | number>(_: T) => void`

virtual 预设点击切换画布展示，不碰编辑器；15 个 chip 全部直接可见、可多行换行。snippet 收进「代码模板」浮层，点击向编辑器追加 `export type CN = <模板>`（N 取现有 `C(\d+)` 最大值 +1，与上文之间保证一行空行），此后代码即事实源

## 编辑器管线（EditorService + EditorDrawer + MonacoEditor）

- monaco 只提供编辑器表面与语法高亮；诊断波浪线（350ms 快速 check → markers）、hover、补全全部由 6.0.3 分析 worker 的 provider 供给，编辑器口径与语义引擎口径天然一致（同一 checker），Footer 明示引擎为 TypeScript 6.0.3
- 提交管线：键入 → 300ms 防抖写 IndexedDB → 1.2s 防抖提交分析（代码 + 激活的 virtual 预设）；virtual 预设切换立即触发分析；分析请求带单调 ticket，过期结果丢弃
- 编辑器光标落在某个导出声明范围内时，画布对应矩形高亮（双向联动保留）
- **抽屉形态**：桌面内联右栏，左缘手柄拖宽（320px 至视口 70%，宽度写 localStorage）、标题栏按钮收起为细轨；窄屏（< 1024px）转为固定定位抽屉，从右侧浮出盖住画布，右缘把手开合，收起即整体移出屏幕

## 持久化与分享

- IndexedDB 文档记录 `{ code, languageId, presets?, updatedAt }`，键入 300ms 防抖保存；`presets` 为激活的 virtual 预设标签数组（envelope v2 新增，老存档缺省为空）
- 分享 hash `#code/v1/<lz-string payload>`，envelope `{ languageId, code, presets? }`；恢复优先级 URL hash > IndexedDB > 示例代码；`Cmd / Ctrl + S` 复制带内容链接 + toast；locale 不进 hash
- i18n 与 v1 相同：中英字典 + mobx locale + localStorage，[ADR-0008](../adr/0008-i18n-自实现轻方案.md)

## 体积与首屏

- 单一 TypeScript 资源（6.0.3，随分析 worker 打包，gzip ~1.6MB）+ lib d.ts raw 资产；monaco 只装 editor core 与语法高亮，不带 TS worker（[ADR-0015](../adr/0015-统一单一-typescript-资源-6.0.3.md)）
- 产物从 tsgo 时期的 61MB 回落到 ~15MB；分析 worker 常驻 LanguageService 增量更新，无 wasm 重复 instantiate 成本
