# 功能技术设计

本文是实现层的技术方案，覆盖语义引擎、布局引擎、渲染、编辑器管线、持久化与分享。架构分层与 IR 契约见[工程架构设计](../architecture/工程架构设计.md)，行为验收见[产品功能定义](../product/产品功能定义.md)。方案中标注「实证」的事实来自对 TypeScript 3.5–7.0 的本地编译器验证

## 语义引擎（adapters/typescript，运行于 worker）

### 运行时

typescript pin 5.9.x + @typescript/vfs 起 LanguageService。TypeScript 7 起是 native 实现、JS compiler API 不再存在（实证：`typescript@7.0.2` 包内只有 tsc 二进制包装），JS 版本线终结于 5.9.3 / 6.0.x —— 所以 pin 是被迫的长期决策，页面 Footer 明示所用 TS 版本，见 [ADR-0002](../adr/0002-技术栈-tanstack-start-与-typescript-单版本-pin.md)

lib.d.ts 通过 [@typescript/vfs](https://www.npmjs.com/package/@typescript/vfs) 的 `createDefaultMapFromCDN` 按需拉取，localStorage + lz-string `compressToUTF16` 缓存（vfs 内建能力）。编译选项：`strict: true`、`noEmit: true`，永不 emit、永不执行用户代码

### export 提取规则

遍历 SourceFile 顶层语句，纳入带 `export` 修饰符的 type alias / interface：

- 带类型参数的声明，仅当所有参数都有默认值时纳入（此时裸引用合法，等价于默认实例化 —— TS 2.3 起，[release notes](https://www.typescriptlang.org/docs/handbook/release-notes/typescript-2-3.html)）
- `export { X }` 转导出、`export *`、enum、class：期 1 不纳入，是已声明的边界

### 关系分类

对每对导出 (A, B) 双向调用 `checker.isTypeAssignableTo`（TS 5.4 起公开 API，[PR #56448](https://github.com/microsoft/TypeScript/pull/56448)）：

- 双向可赋 → `equal`（等价类合并）
- 单向 → `subset` / `superset`
- 双向不可赋 → 进入相交性判定

### 相交性判定与「重叠未知」

判断 A & B 是否为空，统一走 `checker.isTypeAssignableTo(intersection, checker.getNeverType())`，不用 `TypeFlags.Never` 直接判 —— TS 3.9 的判别属性归约是 lazy 的（[PR #36696](https://github.com/microsoft/TypeScript/pull/36696)），Type 对象的 flags 不含 Never 但实际已是空集，直接看 flags 会漏判一整类（实证）

checker 归约覆盖：原语/字面量冲突（TS 3.6，[PR #31838](https://github.com/microsoft/TypeScript/pull/31838)）、含字面量判别属性的对象冲突（TS 3.9）、模板字面量与不匹配字面量。不归约的已知情形：`{a: string} & {a: number}`（[#18210](https://github.com/microsoft/TypeScript/issues/18210)）、互斥模板字面量（[#60446](https://github.com/microsoft/TypeScript/issues/60446)）

判定流程：交集判空为真 → `disjoint`；为假且能找到公共原子 → `overlap`；两者都给不出 → `unknown`（「重叠未知」），渲染为虚线态。教学工具宁可说不知道，不画可能错的图

### any 与 error type

`TypeFlags.Any` 命中则该导出不进集合模型、触发悬浮 badge。拼写错误产生的 error type 在 flags 上与 any 不可区分（内部 `intrinsicName === 'error'`），区分手段是 diagnostics：存在语法/语义错误时管线在编辑器阶段就已拦截（不会提交分析），所以 worker 内遇到 Any flag 即当真实 any 处理

### 固定基底域

`unknown` 切成互不相交的固定 Domain，构成底图：

| Domain      | 说明                                          |
| ----------- | --------------------------------------------- |
| `string`    | 含字面量、模板字面量细分                      |
| `number`    | 含数字字面量                                  |
| `bigint`    |                                               |
| `boolean`   | 内部即 `true` / `false` 两点                  |
| `symbol`    |                                               |
| `null`      | 单点域                                        |
| `undefined` | 单点域；`void` ≈ 此域 + 偏差标注              |
| `object`    | 内部子域：`function` / `array` / plain object |

跨域组合：`unknown` = 全部域；`{}` = 全部域减 `null`、`undefined`。这些映射写死在 TS adapter 的 universe 描述里

### 类型 → 原子分解

每个导出分解为原子集合：

1. **整域覆盖**：对每个 Domain 的代表类型 `D` 检查 `isTypeAssignableTo(D, T)`，为真则 T 覆盖整个域
2. **字面量点**：union 展开后收集字面量成员（`checker` 的 union types + literal flags）
3. **域内细分**：覆盖不了整域、又与该域相交的部分（结构对象、函数签名、模板字面量），成为域内细分区域；同域细分两两做关系分类 + 判空，得到「动态原子」

同域细分数量上限 6（超限 toast 提示删减导出，见 [ADR-0004](../adr/0004-可视化-无降级视图的全-euler-策略.md)）；导出总数上限 12

### 偏差标注（Deviation）

TS 偏离纯集合语义的点，以角标 + hover 说明呈现：

- `any`：唯一被移出图外的（badge）
- 方法参数双向协变：`Kennel`/`DogKennel` 等价合并时挂标注 —— 这是教学演示用例的一部分
- `void`：视为 `{undefined}` + 函数返回值特例说明
- 后续语言的偏差点走同一机制

### 类型文本展开

hover 展示的展开文本：`checker.typeToString(type, undefined, TypeFormatFlags.InTypeAlias | TypeFormatFlags.NoTruncation)` —— 抑制顶层 alias 名、解除 160 字符截断；嵌套 alias 仍打印名字（TS 无公开递归全展开 API，实证），对象结构辅以一层 `{ [K in keyof T]: T[K] }` 展开。`noErrorTruncation` 只影响错误消息、管不到 hover（[#26238](https://github.com/microsoft/TypeScript/issues/26238)），自持 worker 自己调 typeToString 不受此限

## 布局引擎（core/layout）

### 固定底图

各 Domain 有确定性的锚点坐标与基础形状（手工调优的常量表，画布固定尺寸使它可以是绝对坐标）。底图永不因用户输入移动 —— 用户编辑代码时只有轮廓层变化，这是教学工具与炫技 demo 的分界

### 轮廓计算

用 [bubblesets-js](https://github.com/upsetjs/bubblesets-js)（MIT、零依赖 TS、upsetjs 维护线）：每个导出的 Region 原子对应的锚点 `pushMember`，其余锚点 `pushNonMember`，输出 SVG path 的收腰包络。BubbleSets 论文的设计初衷就是「固定主可视化 + 集合轮廓 overlay」（[原论文](http://vialab.science.uoit.ca/portfolio/bubblesets)），本方案是它的标准用法；AntV G6 v5 内置同款算法可作生产背书

### 布局不变量（假交集防护）

bubblesets 轮廓间的视觉交叠是各自独立计算涌现的、不携带语义。防护不变量（[ADR-0010](../adr/0010-布局不变量-假交集防护.md)）：

1. 语义相交的两个 Region → 共享原子的锚点相邻放置
2. 语义不相交的两个 Region → 两者锚点之间放障碍点
3. 渲染前自检：对每对 entity，比较轮廓 path 的几何相交结果与 RelationMatrix，冲突时开发态直接抛错、生产态标记该对为「渲染降级」并打日志

轮廓平滑会削角，锚点半径预留 padding（成员点半径 × 1.5）防止轮廓切割成员

### 嵌套与等价

包含关系靠原子集天然成立（子集的锚点是父集锚点的子集），轮廓间视觉 padding 通过给内层集合更小的锚点半径实现；等价类合并为单一 Region、标签并列

### 双向协变引起的包含环与塌缩（实现期发现）

方法双向协变使 assignability 失去传递性，这带来两个已实证的后果：

1. **membership 需要不动点闭包**。witness ⊆ A、A ≡ B 时 witness ⊄ B 可能发生（bivariance 造成），裸 witness 查询产出的 cells 会违反「等价 ⟹ cell 集合相等」不变量，adapter 在实体级包含边上做不动点闭包后不变量按构造成立（`src/core/set-model/types.ts` 的 Cell 注释已约定）
2. **包含环会整体塌缩为一个等价类**。方法语法与函数属性语法同签名互相可赋值，若同一批导出里混写同名成员的两种语法，会形成 `KennelF ⊆ DogKennelF ⊆ DogKennelM ⊆ KennelM ⊆ KennelF` 这样的环；集合语义下 ⊆ 环意味着全部相等，四个类型合并为一个 `≡` 区域。这是对 TS 不健全性的忠实呈现（不是 bug），本身可作为进阶演示；要展示「方法合并 vs 属性嵌套」的对照，示例须让两组用不同成员名（教学用例已按此写法）

## 渲染（views/EulerCanvas）

SVG 分层结构，自下而上：

1. **底图层**：Domain 基础形状 + 标签，静态
2. **never 点阵层**：`∅` 字形的 SVG `<pattern>` 平铺（原生 pattern tiling，浏览器合成器处理，无逐点 DOM 节点）—— 画布不可缩放使 pattern 分辨率固定，不需要 SDF 动态精度，见 [ADR-0005](../adr/0005-never-点阵与-any-悬浮-badge.md)
3. **区域轮廓层**：每个导出一条 bubblesets path，粗描边（3px）+ 淡色半透明花纹填充（`<pattern>`：点阵/斜线/网格按嵌套层级轮换）
4. **标签层**：区域名字 + 偏差角标，碰撞避让
5. **overlay**（HTML，SVG 之外）：any badge、toast、hover 卡片

配色出自 semi-color-tag 色板（[ADR-0009](../adr/0009-色彩系统-semi-color-tag.md)）：`-0` 档做花纹底、`-1`/`-2` 档做半透明填充、`-3` 档做实色描边；Domain 有固定色相映射，用户导出按声明序从剩余色相轮转

### 呼吸动效

描边波动：SVG `feTurbulence + feDisplacementMap` 滤镜作用于轮廓描边层，`baseFrequency` 低频、位移幅度 3–6px，CSS 动画驱动 seed 缓慢变化。约束：仅描边层挂滤镜（填充不动）、标签层不挂；页面不可见时暂停（`visibilitychange`）；`prefers-reduced-motion` 时移除动画保留静态形状。滤镜性能不达标时的候补是 stroke-dashoffset 缓动，牺牲「波动」保留「流动」

### any badge

HTML 悬浮元素（不在 SVG 内）：橙红底、白字、圆角、`rotate(-15deg)`、`box-shadow` 投影（图内集合无投影，层级差全靠这一处对比）。默认位置横跨画布与编辑器分栏边界中间偏上；pointer 事件实现拖拽，位置写 IndexedDB。hover 展开说明卡片 + 当前解析为 any 的导出清单

### 画布尺寸

固定渲染尺寸（viewBox 常量），容器按可用宽度等比缩放展示但不提供缩放交互；最小宽度 360px，视口更窄时出现横向滚动

## 编辑器管线（views/EditorPane + EditorService）

- monaco-editor ESM 集成，vite `?worker` 方案加载 editor worker 与 TS worker（[官方 vite 样板](https://github.com/microsoft/monaco-editor/blob/main/samples/browser-esm-vite-react/src/userWorker.ts)；monaco 0.55 起 worker deep import 需写全 `.js` 扩展名，[vite#21426](https://github.com/vitejs/vite/discussions/21426)）
- `typescriptDefaults.setCompilerOptions({ strict: true, ... })`，诊断由 monaco TS worker 原生提供
- 提交管线：内容变更 → 取 monaco markers → 无 error 级 marker 则起 1.2s 定时器 → 到期无新变更则 `AnalysisService.submit(code)` → worker 返回 IR → SetModelService 更新 → 画布重绘
- monaco 体积（实测 gzip：editor ~950KB + TS worker ~1.5MB）：编辑器组件懒加载（路由级 code split），加载中占位骨架

## 持久化（PersistenceService）

IndexedDB，[idb](https://www.npmjs.com/package/idb) 薄封装（Promise 化，无额外抽象），见 [ADR-0006](../adr/0006-持久化-indexeddb-与分享编码.md)：

- db `typarium` v1，store `workspace`：单记录 `{ id: 'current', code, updatedAt }`；store `settings`：kv（`locale`、`anyBadgePosition`）
- 写入时机：编辑停顿 500ms 防抖 + `visibilitychange`（hidden）
- 启动恢复顺序：URL hash 内容 > workspace 存档 > 默认示例代码

## 分享（ShareService）

- hash 格式：`#v1/<payload>`，payload = `compressToEncodedURIComponent(JSON.stringify({ code, lang: 'typescript' }))`（lz-string，输出为 URL-safe Base64 变体字符集；TS Playground 同款方案，[Playground 手册](https://www.typescriptlang.org/_playground-handbook/url-structure.html)）
- 版本号在前缀里，将来 schema 变更加 `#v2/` 平滑解码
- `Cmd / Ctrl + S`：preventDefault → 生成带内容链接 → 剪贴板 → toast；Header 分享按钮提供「带内容 / 仅站点」两项
- locale 不进 hash（个人偏好不强加给接收者）

## i18n（SettingsService + i18n 字典）

自实现轻方案（[ADR-0008](../adr/0008-i18n-自实现轻方案.md)）：`Record<MessageKey, string>` 两份字典 + mobx observable locale + `t(key)` 函数。默认 `navigator.language`，切换写 IndexedDB。放弃 i18n 框架的代价：无复数/插值 DSL（模板函数覆盖）、无懒加载命名空间（两语言字典体积可忽略）

## 预设按钮（PresetService）

预设描述符在 adapter 里（`presets: PresetDescriptor[]`），点击 = 通过 EditorService 插入/删除真实代码行（如 `export type _object = object`，行尾 `// preset` 注释标记来源）。代码文本是唯一事实源：按钮选中态由解析当前代码得出，无独立状态

## 体积与首屏

- monaco、分析 worker、bubblesets 全部懒加载；首屏只有布局骨架 + 底图 SVG（纯静态，SSG 预渲染）
- lib.d.ts CDN 拉取 + 本地缓存（vfs 内建）
- 预算：首屏可交互 < 1s（不含 monaco），编辑器就绪 < 4s（冷缓存、普通网络）
