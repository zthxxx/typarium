# ADR 0016 —— 并集穷尽与重叠带布局

- 状态：已采纳
- 日期：2026-07-16

## 背景 Context

[ADR-0012](./0012-可视化范式-矩形包含布局.md) 的初版规则暴露出两个语义缺陷（用户第四轮反馈）：

1. **并集穷尽未检测**。`C3 = string | number` 且 `C1 = string`、`C2 = number` 都在展示时，C3 ≡ C1 ∪ C2 —— C3 内部没有「其他类型」，但布局仍给实体容器一律 +1 空槽，画出了不存在的剩余区域；三个子集时还会因「奇数向上取偶」出现 2×2 留空格
2. **多父包含只挂单亲**。`C1 = string ⊂ C2 = string | number` 且 `C1 ⊂ C3 = string | boolean` 时，包含森林给 C1 只挂最早的最小超集（C2），画面上 C3 不包含 C1 —— 与集合事实相悖

## 决策 Decision

**并集穷尽**：引擎为每个实体计算 `coveredBySubsets`（P ⊆ ∪ 全体真子集，⊇ 由构造成立；用探针文件合成 `__Union = __Ei | __Ej` 别名后一次 assignability 判定）。布局对穷尽容器改规则：槽位数 = 子类数精确值，无 other 槽、无向上取偶，因子分解 cols ≥ rows 差最小（2→2×1、3→3×1、4→2×2）—— 子项完整铺满父容器

**多父与重叠带**：包含结构由森林升级为 DAG，parent = 全部最小真超集。双父节点的两个父类构成「重叠对」：在网格中占两个水平相邻格、各自向对方延伸（左父 [0, 0.62W]、右父 [0.38W, W]），共享子类布局在重叠带 [0.38W, 0.62W] 内，独占子类留在各自独占区。半透明填充在带内自然叠色，恰好是交集区域的视觉表达 —— 矩形范式下的受限 Euler 重叠

**表达力边界（明确保留）**：≥3 个父、跨容器父等复杂格局回退单亲行为并写 warning；两个集合相交但交集内没有任何被展示的实体时，仍不画重叠（无信息可画）。一般化的矩形 Euler 布局是 NP 难问题，本产品只解教学高频形态

## 代价与后果 Consequences

- 穷尽判定每次分析多一遍探针文件更新与 O(N) 次 assignability 查询，LanguageService 增量下开销可忽略
- 重叠带内的子集矩形同时属于两个父，tooltip 的包含栈会正确列出双父（stackAt 按几何包含收集，天然成立）
- 「无空余区域」的字面语义只在穷尽容器成立；非穷尽容器的空槽仍然是「其他类型」的诚实表达
- 含重叠对的行因 unit 宽度差异可能产生行尾余格，视为布局妥协记 warning，不影响语义

## 备选方案 Alternatives

- 在渲染层做通用矩形 Euler 求解 —— 否决：一般情形不可解且布局不稳定，违背确定性不变量
- 穷尽判定放在布局层（对关系矩阵做集合并推理）—— 否决：矩阵只有两两关系，`P ⊆ S₁∪S₂` 无法从 `P ⊅ S₁ ∧ P ⊅ S₂` 推出，必须问 checker
- 多父子集在每个父内各画一份（复制节点）—— 否决：同一集合出现两次违背「等价 ⟹ 同一区域」的既有语义，且复制后两份的子层级会指数膨胀
