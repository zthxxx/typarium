# ADR 0020 —— 启动管线显式化与缓存优先渲染

- 状态：已采纳
- 日期：2026-07-22

## 背景 Context

冷启动实测：产物两大块 analysis.worker 3.9MB（typescript 编译器 + 全量 lib.d.ts 被 rolldown 内联，源码注释宣称的 lazy chunks 并未发生）与 monaco 3.5MB 同时抢带宽；modulepreload 只覆盖入口三个小 chunk，ClientApp → worker → monaco 串行发现。首帧后画布长时间空白：`bootstrapContent` 立即触发分析，但 analyze 在 worker 内等待全量初始化，期间 preset 点击排队且无反馈。目标：首次打开「加载到画布出现计算结果」提速 2x，并以真实进度条呈现启动环节

## 决策 Decision

1. **画布优先**：monaco 延后到首次分析落地（或短兜底超时）再加载，编辑器 3.5MB 不与语义引擎抢首屏带宽
2. **lib 拆出与瘦身**：lib.d.ts 移出 worker chunk，构建期用 TypeScript printer 剥离注释（quickInfo / inlineQueries 只展示类型文本，无功能损失），按 compilerOptions 传递闭包收集，产出内容寻址的 JSON 静态资源；worker 以 fetch 流式加载，字节进度真实可观测
3. **瀑布拉平**：构建后处理向 HTML 注入 ClientApp 与 worker chunk 的 modulepreload
4. **缓存优先渲染**：last-good `AnalysisResult` 连同 key（代码 + presets + engineLabel）存 IndexedDB；启动时 key 精确命中即刻上屏（布局是纯函数），引擎就绪后重验证替换。key 不命中（含 share 链接首开）走进度条路径
5. **BootService**：observable 的阶段状态（引擎下载 / 引擎初始化 / 内容恢复 / 首次分析），进度=已完成阶段数 + 下载字节分数；引擎内部阶段由 adapter 通过 `onBootProgress` 上报，view 只渲染
6. **验收口径**：「可用」= 画布出现计算结果；playwright + CDP 冷启动测量脚本先立基线，2x 以脚本数字判定

## 代价与后果 Consequences

- 缓存优先渲染触碰「画布不呈现可能说谎的数据」不变量：key 必须含引擎版本且精确匹配，命中的结果对该输入曾为真；ATA 依赖远端 typings 的代码存在重验证后被替换的可能，期间 footer 显示重验证状态
- monaco 延后让桌面用户短暂看到编辑器骨架；lib 注释剥离使 hover 失去内置库 JSDoc 文档（当前功能本就不展示，留构建开关可回退）
- worker 预热 fetch 与 `new Worker` 的缓存共享依赖 immutable hashed 资产，上线前需网络面板验证无双下载

## 备选方案 Alternatives

- Service Worker 预缓存全站 —— 否决：引入 SW 生命周期复杂度，且对首次访问无效，正是本次要优化的场景
- 伪进度条（时间函数动画）—— 否决：产品要求真实进度；阶段完成数 + 字节分数都可真实取得
- worker 内嵌 lib 保持现状、仅剥注释 —— 否决：失去与 worker 脚本解析并行的下载窗口，也拿不到字节进度
