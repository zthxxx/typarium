# ADR-0005 never 点阵与 any 悬浮 badge

- 状态：已采纳
- 日期：2026-07-16

## 背景 Context

`never`（空集）与 `any`（集合论之外）是 Euler 图画不了的两个极端，需要专门的视觉语言。前期设计曾担心 never 多点画法会被初学者读成「成员」，需求方给出了更强的方案：点阵

## 决策 Decision

### never = ∅ 点阵

以 `∅` 字形的点阵均匀铺在所有集合区域内（含空画布）—— 类比 Photoshop 里透明背景显示为浅色棋盘格：点阵在视觉上是「底」，表达「空集无处不在、被所有集合包含」，而单个点又保留「空集是一个点」的直觉

实现用 SVG `<pattern>` 原生平铺，放弃手写 SDF：画布不可缩放（固定 viewBox），pattern 分辨率固定，浏览器合成器直接处理平铺，性能与 SDF 等价而实现成本低一个量级。点阵透明度低（视觉为底纹），hover 有说明

### any = 可拖拽悬浮 badge

HTML 悬浮元素：橙红、圆角、斜置约 15°、**带投影阴影** —— 图内所有集合无投影，唯一的投影表达「不在同一层级」。默认位置横跨画布与代码编辑区的边界中间偏上（视觉上宣告「它不在画布里」），用户可拖到屏幕任意位置（类似客服悬浮球），位置持久化。hover 说明 any 不属于集合范畴 + 当前解析为 any 的导出清单

## 代价与后果 Consequences

- 点阵作为全画布底纹，与集合花纹填充叠加时需要控制对比度层级（点阵最淡），设计 token 里定死透明度档位
- SVG pattern 在不同 DPR 屏幕上的字形清晰度需要一次实机验证；不达标的候补是把 `∅` 换成纯几何点 + 环
- badge 是 HTML overlay，不参与 SVG 导出（将来若做「导出分享图」功能需要单独合成）

## 备选方案 Alternatives

- never 画成每个集合内一个点（原始草图方案）：点与字面量单元素集难以区分，且「空集在所有集合中」的语义只能靠多个孤立点暗示，表达力弱于点阵，否决
- SDF 渲染点阵：画布可缩放时才有精度收益，本产品画布固定，成本不划算，否决
- any 画成图内特殊区域：any 双向可赋值的悖论没有几何对应（既是全集又是子集），任何图内画法都在说谎，否决
