# ADR-0010 布局不变量：假交集防护

- 状态：已采纳
- 日期：2026-07-16

## 背景 Context

轮廓算法选用 bubblesets（[ADR-0003](./0003-语义引擎-固定基底域与动态原子.md) 的渲染配套）。BubbleSets 的每条轮廓独立计算，两条轮廓在视觉上交叠是布局「涌现」的结果，不携带任何语义 —— 这是调研确认的头号坑：布局稍有不慎，图会画出语义上不存在的交集，教学工具画错图等于负价值

## 决策 Decision

把「视觉拓扑 = 语义拓扑」定为布局层的硬不变量：

1. 语义相交的两个 Region → 它们共享原子的锚点相邻放置（共享原子就是同一个锚点，天然相邻）
2. 语义不相交的两个 Region → 双方锚点互为 bubblesets 的障碍点（`pushNonMember`），势场绕开
3. 「重叠未知」关系 → 锚点既不相邻也不强隔离，轮廓可能接近但以虚线边界渲染，语义交由样式表达
4. **渲染前自检**：每帧布局输出后，对每对 entity 做轮廓 path 几何相交测试，与 RelationMatrix 比对 —— 不一致时开发态直接抛错（测试必挂），生产态把该对标记为渲染异常并打日志

## 代价与后果 Consequences

- 自检是 O(N²) 的 path 相交测试，N ≤ 12 且只在布局重算时跑，成本可忽略
- 势场参数（网格精度、能量半径）调不好时可能出现轮廓断裂或吞掉障碍点，自检能捕获但修复靠调参 + `pushEdge` 引导边，这部分是实现期的调优债
- 轮廓平滑（`sample().simplify().bSplines()`）会削角，锚点要预留 padding，否则成员点被切在轮廓外 —— 同样由自检捕获

## 备选方案 Alternatives

- 信任 bubblesets 输出不做自检：调研明确了交叠是涌现的，等价于把正确性交给运气，否决
- 用精确 Euler 求解器保证拓扑：回到 [ADR-0003](./0003-语义引擎-固定基底域与动态原子.md) 否决过的通用求解路线，否决
