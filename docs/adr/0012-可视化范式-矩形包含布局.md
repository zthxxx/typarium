# ADR-0012 可视化范式：矩形包含布局

- 状态：已采纳（推翻 [ADR-0004](./0004-可视化-无降级视图的全-euler-策略.md) 与 [ADR-0005](./0005-never-点阵与-any-悬浮-badge.md) 中的 bubblesets 轮廓 + 固定底图路线）
- 日期：2026-07-16

## 背景 Context

第一版可视化走的是「固定基底域底图 + bubblesets 不规则收腰轮廓」：为了在一张图上同时表达包含、不相交与部分重叠，引入了原子分解、动态判空、假交集防护（[ADR-0010](./0010-布局不变量-假交集防护.md)）一整套机制。浏览器验收后需求方否决了这个形态，给出第二轮明确要求：

- 不希望用不规则包围图形表达包含关系，改为只用圆角矩形的位置编排来表达
- 表达模型收窄为「包含或不包含」两种：TypeScript 类型在本产品的讲授框架里不引入「部分有交集」的概念
- 图形面积与类型的「大小」无关，兄弟集合直接平分容器空间
- 画布没有任何类型时显示为空，不再常驻灰色基底域底图

反转的理由成立：教学讲的是包含链与互斥分支（协变嵌套、Tagged Union 并列），不规则包络带来的视觉复杂度对这两件事没有回报，反而让「底图是什么」「收腰为什么这样长」成为解释负担

## 决策 Decision

1. **只有圆角矩形一种图形**。实体渲染为嵌套或并列的圆角矩形，无面积语义、无不规则轮廓
2. **等价类合并**：相互可赋值的实体合并为一个矩形，类内每多一个成员加一圈内缩边框（多环嵌套），标签并列
3. **包含森林**：等价类之间按 subset 关系建 DAG，parent 取最小真超集（多个最小时取声明序最早保证确定性）；无包含关系的类互为兄弟
4. **递归网格平分**：
   - 容器为隐式画布根时，槽位数 = 顶层类数
   - 容器为实体矩形时，槽位数 = 子类数 + 1 —— 多出的 other 槽留白，表达「容器里除了这些子集还有别的值」
   - 槽位数为 1 直接占满；为奇数（> 1）向上取整到偶数，再做双因子分解 cols × rows，取 cols ≥ rows 且差最小（2→2×1、4→2×2、6→3×2、12→4×3）
   - 子项按声明序 row-major 填格，多余格子留白；格间留 gap、容器内留 padding，让背景从缝隙透出
5. **特殊类型的呈现不变**：`never` = 背景点阵（∅ 纹理，展示时铺满画布背景）、`unknown` = 画布本身（展示时画布边框强调 + 标签）、`any` = 悬浮 badge 移出图外
6. **空画布**：没有任何可绘实体时画布完全为空，基底域底图随范式退役
7. **tooltip 改为堆叠列表**：hover 画布任意点，列出包含该点的实体（外层在前，`名字: 类型文本`），末尾附 `∅ never` 行；这是多类型叠放时唯一的消歧手段
8. [ADR-0010](./0010-布局不变量-假交集防护.md) 假交集防护随 bubblesets 一起退役：矩形范式里视觉交叠不存在，不变量失去对象

## 代价与后果 Consequences

- **表达力的有意舍弃**：真实存在的部分重叠（如 `string | number` 与 `number | boolean` 共享 `number`）被渲染为并列矩形，图上读不出交集。这是需求方确认的表达模型收窄，产品口径是「本工具只讲包含」；遇到追问时 tooltip 的堆叠列表是唯一线索，教学讲稿需自行补位
- 平分面积使「图形大小」失去含义，初学者可能误读 `"foo"` 与 `string` 面积相当 —— 靠嵌套层级表达大小关系，接受
- other 槽规则里「实体容器恒 +1、画布根不加」是对需求描述中两处例子（两元素平分画布 vs A⊃{B,C} 时 B、C 各占三分之一）的统一化，奇数取偶后实际格子占比与「三分之一」的字面描述有出入，以网格算法为准
- 收益是复杂度大幅下降：原子分解、动态判空、bubblesets、布局自检整链路删除，布局引擎从「研究级问题的工程规避」变成一百多行确定性纯函数

## 备选方案 Alternatives

- 维持 bubblesets 轮廓、仅做样式收敛 —— 否决：需求方点名不要不规则包围图形
- treemap 式按基数加权分配面积 —— 否决：类型的「大小」多数无限或不可比，面积语义站不住
- 矩形 + 部分重叠时物理交叠摆放 —— 否决：与「只有包含或不包含」的表达模型矛盾，交叠面积又会被误读为有语义
