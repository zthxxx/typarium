# ADR 0017 —— 双布局引擎：Euler 矩形与 Hasse 图自动切换

- 状态：已采纳
- 日期：2026-07-16

## 背景 Context

[ADR-0016](./0016-并集穷尽与重叠带布局.md) 的重叠带把矩形范式推到了双父同容器的场景，但用户第六轮的 C1..C6 例子暴露了表达力天花板：

```typescript
export type C1 = string
export type C2 = number
export type C3 = string | number | boolean
export type C4 = string | number | null
export type C5 = string | number
export type C6 = number | boolean | null
```

校正后的包含 DAG 中，C2 的最小真超集是 C5 与 C6 —— C5 位于 C3∩C4 重叠带内（深度 2）、C6 是顶层根（深度 1），两个父不在同一容器；同时 C3/C4/C6 两两相交且交集内容各不相同。在「不拆分同名实体矩形」的约束下（复制节点已被 [ADR-0016](./0016-并集穷尽与重叠带布局.md) 否决），轴对齐单矩形无法同时满足这些嵌套与交叠 —— 这是矩形 Euler 图的已知理论边界，不是实现缺陷

## 决策 Decision

新增 **Hasse 图布局引擎**作为复杂格局的自动降级，`computeCanvasLayout` 返回带 `mode` 判别的联合：

1. **切换判据 = 忠实性探测**：构建等价类与包含 DAG 后，收集全部「矩形无法忠实表达」的信号 —— ≥3 个最小真超集、双父跨容器、重叠对配对冲突。零信号 → Euler 矩形（既有 v3 引擎原样输出）；有信号 → 整画布切 Hasse，信号文本进 warnings。能用 Euler 尽量 Euler 是硬约束：判据只在矩形确实画不对时才放行降级
2. **Hasse 语义**：节点 = 等价类（多标签 ≡ 合并语义不变）；边 = 覆盖关系（全部最小真超集，不限父数）；超集在上子集在下；最长路径分层、层内 barycenter 排序
3. **??? 语义延续**：有子节点且未被子集穷尽的实体类，在 Hasse 里同样挂一个显式 `???` 占位子节点 —— 「父下还有其他元素」在两种引擎中表达一致
4. unknown 在 Hasse 中是普通最大元节点；never/any 沿用点阵/badge 分流

四类关系正确性（嵌套、父子层级、父下额外元素、等价）在两种模式下语义等价，只是几何隐喻不同：Euler 用面积包含，Hasse 用上下层级 + 边

## 代价与后果 Consequences

- Hasse 丢失面积隐喻（交集不再是可视区域，而是共享子节点的双入边），教学表达力弱于 Euler —— 这正是「尽量 Euler」的原因
- 两种模式的 hover/tooltip/高亮交互需要各自实现，渲染层分支增加
- 切换是整画布级的：局部复杂就整体切 Hasse，不做混合布局（混合的视觉语法成本远超收益）
- 用户编辑一行代码可能引起模式跳变（从 Euler 跳到 Hasse），这是关系复杂度跨过阈值的诚实反馈，不做平滑伪装

## 备选方案 Alternatives

- 复制同名实体矩形（Riche & Dwyer 式 duplication）—— 已在 ADR-0016 否决：破坏「等价 ⟹ 同一区域」
- 通用约束求解（Penrose 等）—— 否决：不确定性布局 + 重依赖，违背确定性不变量
- Linear diagram 作为降级 —— 第一轮调研中实证任务表现最好，但线段隐喻与本产品的「集合是区域」的教学主张距离更远；Hasse 与包含偏序同构，父子层级一眼可读，更贴合「关系正确性」的核心诉求
