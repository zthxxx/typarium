# ADR-0014 预设双轨与编辑器抽屉

- 状态：已采纳
- 日期：2026-07-16

## 背景 Context

第一版预设只有 8 个按钮，交互是「点击 = 向编辑器插入/删除真实代码行」，代码文本是唯一事实源。第二轮需求把预设目录扩到 26 项并给出两类截然不同的行为预期：简单类型点完不改代码、直接上画布；代码模板点完要在编辑器里插入可编辑的声明行。同时编辑器从固定右栏升级为可拖宽、可收起、窄屏浮出的抽屉

## 决策 Decision

1. **预设双轨**：
   - **virtual 预设**（8 基本 + 4 内置 + 3 常见，共 15 项）：点击切换画布展示状态，不触碰编辑器代码；被激活的项以 `{ name, typeText }` 虚拟类型进入分析（`origin: 'preset'`），再点取消
   - **snippet 预设**（11 项代码模板）：点击向编辑器追加 `export type CN = <模板>`，N 自动递增（扫描现有 `export type C(\d+)` 取最大值 +1），与上一段内容之间保证一行独立空行；插入后代码就是唯一事实源，与预设按钮再无绑定
2. **可见性分层**：virtual 预设必须全部直接可见，允许多行换行，禁止收进浮层；snippet 预设收进「代码模板」浮层（点击展开，外点关闭）。目录顺序 = 分类顺序（基本 → 内置 → 常见 → 模板），不渲染分组样式
3. **any 常驻警示色**：any 的 chip 无论激活与否都用暖红色调（未激活淡红边框底色、激活红底白字），与其他 chip 的品牌蓝体系区分 —— 它是唯一会打破集合推理的选项，视觉上先行预警
4. **编辑器抽屉**：
   - 桌面（≥ 1024px）：内联右栏，左缘拖拽手柄调宽（下限 320px、上限视口 70%，宽度写 localStorage），标题栏收起按钮把它收成一条细轨，点细轨恢复
   - 窄屏（< 1024px）：编辑器变成固定定位的抽屉，从右侧浮出盖在画布上，收起时整体平移出屏，屏幕右缘保留一个把手按钮开合；不再采用上下堆叠布局
5. **持久化与分享升级**：IndexedDB 文档与分享 envelope 增加 `presets` 字段（激活的 virtual 预设标签数组）；字段可选，v1 老链接与老存档照常解码，`presets` 缺省为空

## 代价与后果 Consequences

- 事实源从「代码文本唯一」变为「代码 + virtual 预设状态」双源，分享与持久化都要携带两者；换来的是画布可以零代码演示（点几个 chip 就开讲），这是教学场景的直接收益
- virtual 预设的展示名与类型表达式解耦（如 `Array<T>` 展示、`Array<unknown>` 参与分析），名字里的 `T` 是教学示意，语义上是 unknown 实例化 —— 讲稿需要知道这一点
- snippet 的 C1/C2 命名可能与用户已有的同名类型冲突，取最大编号 +1 只保证不与既有 CN 冲突，不做全局重名检查（重名时编辑器诊断会直接报错，交给用户处理）
- 窄屏抽屉盖在画布上，打开时画布被遮挡 —— 需求方点名要这个形态（替代「堆叠在画布下面要滚动才能看到」的旧行为）

## 备选方案 Alternatives

- 所有预设统一走插代码（v1 方案延伸） —— 否决：需求方明确 virtual 类点击不得改动编辑器内容
- 所有预设统一走 virtual 状态 —— 否决：模板类的价值就在于生成可继续编辑的真实代码
- 窄屏保持上下堆叠 —— 否决：需求方点名折叠必须「直接收回去」，堆叠形态需要滚动查看被点名淘汰
