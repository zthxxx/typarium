# ADR-0009 色彩系统 semi-color-tag

- 状态：已采纳
- 日期：2026-07-16

## 背景 Context

集合渲染需要：N 个集合各有可区分的边框色与淡色半透明背景、粗实边框、清新配色。需求方指定参考 Semi Design 的 tag 色板（semi-color-tag 系列）。该色板每个色相四档：`-0`（α 0.08）、`-1`（α 0.2）、`-2`（α 0.4）、`-3`（实色）—— 结构正好匹配「淡花纹底 / 半透明填充 / 实色粗描边」的分层需要

## 决策 Decision

1. 集合区域用 semi-color-tag 色板：`-0` 档做花纹底、`-1` / `-2` 档做区域半透明填充（按嵌套层级）、`-3` 档做实色描边
2. Domain 固定色相映射（string→blue、number→amber、boolean→green、bigint→lime、symbol→violet、null/undefined→grey、object→indigo、function 子域→purple、array 子域→light-blue）；用户导出按声明序从剩余色相轮转
3. Chrome 层（header/footer/按钮）用品牌色：TS 蓝 `#3178C6`、JS 黄 `#F7DF1E`；any badge 橙红（red/orange 系实色档）
4. 花纹（点阵 / 斜线 / 网格按嵌套层级轮换）与颜色并列作为区分手段 —— 重叠区域色相接近时靠花纹分辨，同时对色弱友好

## 代价与后果 Consequences

- 色值以 CSS variables 形式进 design token（`--set-color-<hue>-<level>`），与 Semi 组件库无任何依赖关系，只借色值
- 15 个色相扣除 Domain 固定占用后，用户导出可用约 6-7 个色相；超出后轮转复用，靠花纹与标签区分 —— 与导出上限 12 匹配
- 品牌蓝 `#3178C6` 与色板 blue 不同色值，两者场景隔离（chrome vs 集合区域），不混用

## 备选方案 Alternatives

- 纯品牌四色（蓝/黄/橙/白）：撑不起 N 集合区分度，早期讨论已否决
- 自造色板：Semi 色板已经过大规模产品验证的明度/饱和度一致性，自造无收益，否决
