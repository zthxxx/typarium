# ADR 0022 —— 泛型在约束上界实例化显示

- 状态：已采纳
- 日期：2026-07-22

## 背景 Context

无默认值的泛型导出（如 `export type Handler<X> = (value: X) => void`）此前被扫描器直接跳过：泛型是类型构造器而非单一集合，不实例化就画不出来。用户需要泛型本身出现在图上，并能研究「泛型 vs 带 extends 约束的泛型」之间的代数集合嵌套关系

## 决策 Decision

每个类型参数按 **default → constraint → `unknown`** 逐级回退取实例化参数，泛型导出以该「规范代表」参与分析与绘制：

1. 全参数带默认值：维持既有行为（bare 引用按默认值实例化）
2. 其余泛型：探针别名写成显式实例化 `__En = __In<C1, C2, …>`，`T extends C` 取约束文本 `C`，无约束取 `unknown`
3. 实体标签完整复现参数声明，含约束与默认值（`Handler<X>`、`BoxStr<T extends string>`、`WithDefault<T = string>`），tooltip 的 expandedText 展示实例化后的展开文本 —— 族的定义域与其上界代表在图上同时可读
4. 约束文本引用了兄弟参数（`<T, U extends T>`）或默认值互引时，探针别名会报错 —— 沿用 broken-alias 防御：只丢弃该实体，不污染整个结果

选择上界作为规范代表的理由：`T extends C` 的族 `{F<T> | T ⊆ C}` 在协变位置下并集恰为 `F<C>`（上界是族的最大元），逆变位置下 `F<C>` 是族的最小元 —— 两个方向都把「约束如何决定族的边界」变成图上可见的嵌套：`Box<T extends string>` ⊆ `Box<T>`（协变收缩），`H<T>` ⊆ `H<T extends string>`（逆变反转）

## 代价与后果 Consequences

- 单一代表无法表达整个族：逆变泛型在上界处是族的最小集而非「族本身」，教学口径需说明「显示的是族在约束处的实例」；tooltip 的展开文本承担这一说明
- `Co<T> = T | boolean` 这类在 `unknown` 处退化为全集的泛型会显示为 universe 画布框 —— 行为正确但初看意外，属于「上界即 unknown」的直接推论
- 既有「全默认值泛型」的实体标签从 `Name` 变为 `Name<T>`，分享链接与快照的实体 id 随之变化（快照 key 含代码文本，自然失效重算，无迁移成本）

## 备选方案 Alternatives

- 继续跳过无默认值泛型 —— 否决：用户明确需要比较泛型与约束泛型的集合关系
- 以若干采样点实例化（如 never / 约束 / unknown 三点绘成族）—— 否决（本期）：一个导出膨胀成多个实体，画布拥挤且 MAX_ENTITIES 预算被稀释；用户手写具体实例化导出即可达成同样对比
- 语法级特殊图形表示「族」—— 否决：违反「画布只表达包含」的 ADR-0012 范式
