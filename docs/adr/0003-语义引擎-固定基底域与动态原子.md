# ADR-0003 语义引擎：固定基底域与动态原子

- 状态：已采纳
- 日期：2026-07-16

## 背景 Context

把任意 N 个类型的集合关系画成 Euler 图，通用做法是把关系描述喂给 Euler 布局求解器。两个硬障碍：相交性判定在一般情形不可判定（checker 的 never 归约只覆盖部分情况）；任意关系组合不保证存在 well-formed 的 Euler 图画法（n 个圆最多切出 n²−n+2 个区域 < 2ⁿ，[理论边界](https://www.johndcook.com/blog/2024/09/28/limitations-on-venn-diagrams/)）

## 决策 Decision

在语义层收缩问题，放弃通用求解：

1. `unknown` 按 TS 值域切成互不相交的**固定基底域**（string / number / bigint / boolean / symbol / null / undefined / object 及其子域），构成永不移动的底图
2. 用户类型分解为「整域覆盖 + 字面量点 + 域内细分」三类成分；只有域内细分需要动态计算原子，数量设上限（同域 6、总导出 12）
3. 包含关系用 `checker.isTypeAssignableTo` 双向查询；判空统一走 `isTypeAssignableTo(A & B, getNeverType())` —— TS 3.9 判别属性归约是 lazy 的，直接看 `TypeFlags.Never` 会漏判（本地实证）
4. 判定不了的类型对进「重叠未知」一等状态，虚线渲染，不猜测

## 代价与后果 Consequences

- 底图是 TS 专有知识，每个新语言 adapter 要自己定义 universe 切分 —— 这本来就是语言知识应在的位置
- 域内细分的 2^k 原子计算限制了同域复杂对象的数量，超限只能提示删减（配合 [ADR-0004](./0004-可视化-无降级视图的全-euler-策略.md) 的无降级视图决策）
- 「重叠未知」的存在意味着图承认自己有盲区，教学表达上需要解释成本；换来的是图上画出的每条相交/不相交关系都可信

## 备选方案 Alternatives

- venn.js / edeap 通用面积成比例求解：只做近似、不保证拓扑正确、每次编辑全图跳变，且 edeap 是 GPL，否决
- 纯 assignability 不做判空：无法区分不相交与部分重叠，Tagged Union 演示（variant 两两不相交）画不出来，否决
