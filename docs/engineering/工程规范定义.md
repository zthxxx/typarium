# 工程规范定义

这个仓库关心代码质量、基建与测试，规范服务于可扩展性：所有约束都由工具强制执行，不依赖自觉

## 语言约定

- 代码、注释、标识符、commit message、错误信息：工程化英文
- docs/ 是仓库唯一中文区（含 ADR）
- UI 文案走 i18n 字典（zh / en），不算代码注释的例外

## 质量门禁

合入 main 的代码必须全部通过：

1. `pnpm lint` —— ESLint（`@tanstack/eslint-config` 基线）
2. `pnpm check` —— Prettier（`prettier --check`）
3. `pnpm typecheck` —— tsc strict，`noUnusedLocals` / `noUnusedParameters` 等严格项全开
4. `pnpm test` —— vitest 单元测试
5. `pnpm build` —— 生产构建成功
6. `pnpm test:e2e` —— Playwright 端到端

本地由 git hooks 前置拦截，CI 全量复核；两者跑同一套命令，不允许 CI 专属逻辑

## Git hooks

- 工具：simple-git-hooks + lint-staged
- `pre-commit`：lint-staged 对暂存文件跑 eslint --fix + prettier --write；类型检查全量跑 tsc（strict 项目里增量 lint 挡不住类型回归）
- `commit-msg`：commitlint，Conventional Commits（`feat` / `fix` / `docs` / `chore` / `refactor` / `test` / `ci` / `build`）

## 测试策略

分层对应测试形态，测试写在被测对象旁边（`*.test.ts` 同目录）：

- **core（纯函数层）**：优先 property-based testing（fast-check）表达不变量，例如「布局确定性：同输入（含 relations 乱序）输出逐字节一致」「矩形包含：子矩形几何上必须完全落在父矩形 contentBox 内」「等价类合并与声明顺序无关」。手工用例只补 property 覆盖不到的具体回归
- **adapters/typescript**：用真实 typescript 编译器做集成单测（Node 环境直接跑，无需浏览器），教学演示用例全部进用例集：协变/逆变/双向协变/Tagged Union 的预期关系逐条断言
- **E2E（Playwright）**：教学演示用例作为固定回归集 —— 输入代码 → 等待可视化 → 断言 DOM 结构（区域数量、嵌套关系、any badge 出现）+ 截图比对关键场景；持久化（刷新恢复）、分享链接还原、i18n 切换各一条用例
- 截图基线更新必须是显式动作（`--update-snapshots`），diff 出现在 PR 里

## CI（GitHub Actions）

- 触发：push 到 main、PR
- gates 顺序：install → lint → format:check → typecheck → unit test → build → e2e（Playwright 官方容器或 action 装浏览器）
- 部署 workflow 独立：main 构建通过后发布 GitHub Pages（私有仓库不支持 Pages 时保留 workflow，公开后自动生效）

## Git 约定

- 单人项目，main 直推，不留长期分支
- 完成一个步骤并通过验证后即 commit，一个 commit 一件事；messages 遵循 Conventional Commits
- 图片素材走 Git LFS（`*.png` `*.jpg` `*.webp`）；音视频与二进制工具不入库
- 大量文件移动与内容修改不放同一个 commit（rename 检测与 diff 可读性）

## 文档与 ADR 约定

docs/ 目录职责见 [docs/README.md](../README.md)。规则：

- 需求变化先改 [原始诉求](../requirements/原始诉求.md)（唯一事实源），再改产品/设计文档，方向一致后才动代码
- 有争议或有代价的技术决策：先写 ADR 再实现。ADR 格式：状态 / 日期 / 背景 / 决策 / 代价与后果 / 备选方案（含否决原因）；决策被推翻时不删旧 ADR，新 ADR 引用并标注取代关系
- 文档写作遵循 working-writing-style（中文段末不加句号、判断给代价、不写协作过程）

## 依赖管理

- pnpm，lockfile 入库；Node ≥ 22
- typescript 单版本 exact pin 6.0.3（引擎与工具链共用同一份，[ADR-0002](../adr/0002-技术栈-tanstack-start-与-typescript-单版本-pin.md) 原则 + [ADR-0015](../adr/0015-统一单一-typescript-资源-6.0.3.md) 版本），升级需要跑全量教学用例集回归
- 新增运行时依赖需要说明：解决什么问题、体积影响、license（GPL/AGPL 不引入，见调研教训 [ADR-0004](../adr/0004-可视化-无降级视图的全-euler-策略.md) 附注）
